# 图片型高质感PPT生成 Skill

## 使用场景

当用户希望根据方案文档、汇报材料、售前方案或项目建设方案生成高质感PPT时使用本Skill。尤其适用于用户明确表示普通PPT太简单、需要参考图效果、需要图片级页面、需要蓝白科技风或政企数字化风格的场景。

常见触发语包括：

- 根据这个方案做PPT
- 参考这个PPT风格生成
- 你先生成一张我看看效果
- 我要的是图片这种效果
- 用image2生成PPT页面
- 先给我图，后面再转PPT
- 把图片合并成PDF
- 做成蓝白科技风、政企风、数字化平台风

核心工作法：

先定内容，再定风格，再出样张，再批量生成，最后合并交付。

## 总体流程

### 1. 收集输入材料

开始前确认用户是否提供：

- 方案正文或方案文档
- 参考PPT模板或参考截图
- 项目名称
- 客户名称
- 汇报单位
- 日期
- 是否需要公司介绍页
- 最终交付格式：图片、PDF、PPT
- 是否要求页面可编辑

如果用户追求精致视觉效果，优先走“图片型PPT页面生成”路线。如果用户要求可编辑PPT，应说明图片型页面视觉效果最好，但页面元素不可直接编辑。

### 2. 生成PPT目录结构

目录主题控制在4个以内。

推荐结构：

- 01 建设背景与需求理解
- 02 总体建设方案
- 03 核心功能设计
- 04 建设路径与预期成效

完整PPT一般包含：

- 封面页
- 公司介绍页
- 目录页
- 章节标题页
- 内容页
- 尾页

### 3. 输出逐页文案版大纲

如果用户说“我要第一页、第二页这种”，输出逐页文案版，不要只给表格。

每页采用以下格式：

```text
第X页：页面名称

标题：
页面主标题

页面内容：
3到6条短句

页面说明：
这一页讲什么，可作为口播稿

版式建议：
中心圆、卡片、流程线、架构图、对比图等
```

每页文字要少，适合直接放进PPT。

### 4. 先生成样张确认风格

不要直接批量生成完整PPT。先选1页典型内容页生成样张。

优先选择：

- 一期建设重点
- 业务闭环设计
- 合同管理
- 系统总体架构

样张用于确认：

- 整体风格
- 颜色体系
- 页面精致度
- 字体大小
- 用户是否接受图片型PPT
- 卡片、圆形科技元素、图标、底部波浪线、城市剪影等元素是否满意

### 5. 图片生成风格规范

默认风格：

- 16:9 横版PPT页面
- 蓝白科技风
- 浅蓝渐变背景
- 白色圆角卡片
- 深蓝大标题
- 中心科技圆环或模块图
- 蓝色线性图标
- 轻微阴影和高光
- 底部蓝色波浪线
- 右下角淡色城市天际线
- 适合国企、政企客户汇报
- 文字清晰、正式、中文可读
- 每页保留页码
- 不要水印，不要多余Logo

图片生成提示词模板：

```text
请生成一张16:9横版PPT页面图片，蓝白科技风，浅蓝渐变背景，白色圆角卡片，深蓝标题，蓝色线性图标，科技圆环、流程线、轻微阴影和高光，底部有蓝色波浪线，右下角有淡色城市天际线，整体适合国企/政企客户正式汇报。页面文字必须清晰、中文正确，不要水印，不要多余Logo。页面标题为：XXX。页面内容包括：XXX。版式采用：XXX。
```

多页批量生成时明确：

```text
请分别生成8张独立的16:9 PPT页面图片，不要生成拼图。每张图单独输出，风格保持一致。
```

### 6. 分批生成页面图片

不要一次生成全部页面。建议每批最多8张。

推荐批次：

第一批：1-8页  
第二批：9-16页  
第三批：17-24页  
额外批次：公司介绍页或补充页

如果误生成拼图，必须重新生成，并明确要求“单独图片，不要拼图”。

### 7. 页面质量检查

每批生成后检查：

- 标题是否正确
- 页码是否连续
- 公司名是否正确
- 项目名是否正确
- 中文是否有错字
- 是否误带Logo
- 是否误生成拼图
- 关键业务名词是否正确
- 内容是否被AI自行改写
- 风格是否与样张一致
- 是否有水印、按钮、角标等异常内容

重点检查：

- 客户名称
- 公司名称
- 项目名称
- 人名、地名、案例名
- 合同管理、发票管理、资产台账、投决资料等业务名词

### 8. 按反馈修正页面

常见修正方式：

- 公司名、标题、关键错字错误，单独重生成该页。
- 用户要求去掉序号，标题不得出现1.1、1.2等。
- 用户要求内容不变，只允许改风格，不要改文案。
- 用户要求不带Logo，图片中不得出现对应Logo。
- 用户指出案例名错误，必须重生成修正页。

### 9. 整理最终顺序

根据用户要求调整页面顺序。

常见顺序：

1. 封面页
2. 公司介绍页
3. 股东公司介绍页
4. 客户服务页
5. 目录页
6. 章节标题页
7. 正文页
8. 尾页

必须按最终顺序合并PDF或PPT。

### 10. 合并PDF

使用 `reportlab` 将所有图片按顺序合并成PDF。

示例Python代码：

```python
from reportlab.pdfgen import canvas
from reportlab.lib.utils import ImageReader
from PIL import Image
import os

files = [
    "01_封面.png",
    "02_公司介绍.png",
    "03_股东公司介绍.png",
    "04_客户服务.png",
    "05_目录.png",
]

base = "/mnt/data"
paths = [os.path.join(base, f) for f in files]

out = "/mnt/data/方案_完整版.pdf"
first = Image.open(paths[0])
c = canvas.Canvas(out, pagesize=first.size)

for p in paths:
    img = Image.open(p)
    iw, ih = img.size
    c.setPageSize((iw, ih))
    c.drawImage(ImageReader(img), 0, 0, width=iw, height=ih)
    c.showPage()

c.save()
print(out)
```

注意：

- 不要优先用PIL直接保存PDF，部分环境可能缺少JPEG插件。
- `reportlab` 更稳定。
- 每页PDF尺寸按对应图片尺寸设置，避免拉伸。
- 合并前必须确认图片顺序。

### 11. 转成图片版PPT

如果用户需要PPT格式，可将图片逐页铺满PPT。

特点：

- 视觉效果完整
- 页面元素不可直接编辑
- 适合汇报展示

生成逻辑：

- 创建16:9 PPT
- 每页插入一张图片
- 图片铺满整页
- 导出pptx

可用 `pptxgenjs` 实现。


如果生成PPT：

```text

```

## 常见页面类型与版式建议

封面页：左侧大标题，右侧3D数字化平台插画，底部蓝色波浪线。  
目录页：四个目录项，圆形编号，白色卡片。  
章节标题页：大号数字01、02、03、04，中心标题，留白。  
痛点页：五个卡片，每个卡片一个痛点和图标。  
建设重点页：中心圆+三组模块，一个底座、两条主线、两个轻量模块。  
业务闭环页：四条流程线，箭头串联。  
架构页：五层架构，自上而下分层。  
功能页：中心圆+外围功能卡片。  
实施路径页：横向时间轴。  
成效页：六个价值卡片。  
尾页：感谢观看，大号蓝字，简洁背景。

## 风险与注意事项

- 图片生成中文文字容易出错，必须检查。
- 图片型PPT不可编辑，需提前告知。
- 用户要求内容不变时，不要自行改文案。
- 公司名、客户名、项目名错误必须重做。
- 不要出现用户未要求的Logo。
- 不要把多页生成一张拼图。
- 不要过度堆字。
- 不要把一期边界写模糊。
- 不要承诺图片型PPT元素可编辑。

## 一句话总结

先把内容逻辑讲清楚，再用图片生成把页面做漂亮，最后合并PDF或插入PPT交付。
